# Telegram Bot · Intents Catalog

Бот `@Cyfr_work_bot` принимает голос и текст. Поток: Telegram → n8n → Whisper (транскрипция) → GPT (классификация интента) → выполнение → подтверждение пользователю.

---

## Архитектура обработки

```
[Voice/Text] → Whisper → GPT classifier (JSON) → Validator
                                                     ↓
                                            intent + slots + confidence
                                                     ↓
                                           ┌─────────┴─────────┐
                                           ↓                   ↓
                                  confidence ≥ 0.85    confidence < 0.85
                                           ↓                   ↓
                                   Confirm dialog      Clarify dialog
                                   ("точно ли…?")      (уточняющий вопрос)
                                           ↓                   ↓
                                   Airtable write ←────┘
                                           ↓
                                   Reply + link на обновлённый график
```

### Правила взаимодействия

1. **Даже при высокой уверенности** — всегда reconfirm перед записью: "Точно ли я поняла что вы хотите [action]?".
2. **При низкой уверенности** (< 0.85) — задать уточняющий вопрос с вариантами.
3. **Всегда возвращать ссылку** на публичный график после успешного апдейта.
4. **Ошибки классификации** → fallback к `help` интенту + оригинальный текст для анализа.

## Каталог интентов

### 1. `shift_task` — сдвинуть задачу
Перенос одной задачи (или нескольких) на N дней.

**Slots:** `task_id` (или описание), `days` (int, +/-), `cascade` (bool, default=true)

**Примеры:**
- «Демонтаж сдвинь на 3 дня вперёд»
- «Шпатлёвку стен отложи на неделю»
- «Установку дверей начать раньше на 2 дня, остальные задачи не трогать»

**Confirm:** «Сдвинуть «Демонтажные работы» на +3 дня? Остальные задачи сдвинутся автоматически.»

---

### 2. `mark_start` — отметить начало работы
Установить `actualStart` = сегодня (или указанная дата).

**Slots:** `task_id`, `date` (optional, default=today)

**Примеры:**
- «Демонтаж начали»
- «Сегодня запустили шпатлёвку»
- «Покраску начали вчера»

**Confirm:** «Отметить, что «Демонтажные работы» начаты сегодня, 22 апреля?»

---

### 3. `mark_complete` — отметить завершение работы
Установить `actualEnd` = сегодня (или указанная дата).

**Slots:** `task_id`, `date` (optional, default=today)

**Примеры:**
- «Демонтаж закончили»
- «Шпатлёвка стен готова»
- «Покраска завершена вчера»

**Confirm:** «Закрыть задачу «Демонтажные работы» сегодняшней датой?»

---

### 4. `set_actual_dates` — указать фактические даты явно
Явная установка `actualStart` и/или `actualEnd`.

**Slots:** `task_id`, `actualStart` (optional), `actualEnd` (optional)

**Примеры:**
- «Монтаж утеплителя: факт с 5 по 12 июня»
- «Демонтаж реально начали 6 мая»

**Confirm:** «Записать «Монтаж утеплителя»: факт 05.06 — 12.06?»

---

### 5. `add_delay` — указать задержку и причину
Пометить задачу как задержанную с текстовым комментарием.

**Slots:** `task_id`, `reason` (text), `days` (optional)

**Примеры:**
- «Потолки задерживаются на 4 дня, не приехал материал»
- «Шпатлёвка стоит — ждём согласование цвета»

**Confirm:** «Записать задержку «Шпатлёвка стен» — причина: ждём согласование цвета?»

---

### 6. `add_subcontractor` — добавить субподрядчика
Присвоить задаче субподрядчика.

**Slots:** `task_id` (or multiple), `subcontractor_name`, `phone` (optional)

**Примеры:**
- «Электрику делает бригада Ахмеда, телефон плюс семь…»
- «Сантехнику отдали Марату»

**Confirm:** «Субподрядчик «Ахмед» присвоен задачам: электрика, розетки, освещение?»

---

### 7. `query_period` — запрос задач на период
Получить список задач по произвольному периоду.

**Slots:** `period_start`, `period_end`, `status_filter` (optional: all/active/overdue)

**Примеры:**
- «Что на этой неделе?»
- «Задачи на май»
- «Что будет через месяц?»
- «Покажи, что нужно сделать до пятницы»
- «Задачи на следующий квартал»

**Reply:** список задач с датами + ссылка на отфильтрованный view.

---

### 8. `query_overdue` — просроченные задачи
Показать задачи с `planEnd < today` и `actualEnd` пустым.

**Slots:** — (без аргументов)

**Примеры:**
- «Что просрочено?»
- «По каким задачам отставание?»

**Reply:** список + дней просрочки + суммарная критичность.

---

### 9. `query_status` — общий отчёт
Снимок прогресса проекта: % выполнено, закрыто/всего, задержки, ближайшие дедлайны.

**Slots:** — (без аргументов)

**Примеры:**
- «Как дела?»
- «Статус проекта»
- «Короткий отчёт»

**Reply:** шорт-репорт + ссылка на график.

---

### 10. `add_note` — добавить заметку
Свободная заметка к задаче (не меняет даты).

**Slots:** `task_id`, `note` (text)

**Примеры:**
- «К шпатлёвке: заказчик попросил отшкурить торец получше»
- «По плитке — проверить уклон в сан.узле 2»

**Confirm:** «Добавить заметку к «Шпатлёвка стен»: «заказчик попросил отшкурить торец получше»?»

---

### 11. `list_tasks` — список всех задач
Полный список (с пагинацией или ссылкой на web).

**Slots:** `section_filter` (optional), `stage_filter` (optional)

**Примеры:**
- «Покажи все задачи»
- «Что у нас по стенам?»
- «Все задачи на этапе чистовых»

**Reply:** инлайн список (top 10) + ссылка «смотреть полностью».

---

### 12. `help` — справка
Список возможностей + примеры команд.

**Slots:** —

**Примеры:**
- «Что ты умеешь?»
- «Помощь»
- «/start», «/help»

**Reply:** статичное меню с inline-кнопками:
- 📅 Статус проекта
- ⏰ Просроченные
- 🗓 На этой неделе
- 📋 Все задачи
- 🌐 Открыть график в браузере

## Confirmation flow · шаблоны

### Высокая уверенность (confidence ≥ 0.85)
```
👉 Точно ли я поняла что вы хотите [краткое описание действия]?
   [✅ Да, применить]  [✏️ Изменить]  [❌ Отмена]
```

### Низкая уверенность (confidence < 0.85)
```
🤔 Я не до конца поняла. Вы имели в виду:
   [вариант 1]
   [вариант 2]
   [другое — напишите подробнее]
```

### Ambiguous task_id (нашли несколько)
```
🔍 Нашла несколько подходящих задач. Уточните:
   [1.1 Возведение конструкций из ГКЛ]
   [1.5 Шпатлёвка стен]
   [1.8 Покраска стен]
```

### После успешного апдейта
```
✅ Готово. График обновлён.
🔗 https://cyfr-schedule-app.vercel.app/
```

## Response schema · JSON от GPT classifier

```json
{
  "intent": "shift_task",
  "confidence": 0.92,
  "slots": {
    "task_id": "5.3",
    "task_query": "демонтаж",
    "days": 3,
    "cascade": true
  },
  "clarify_question": null,
  "original_text": "Демонтаж сдвинь на три дня вперёд"
}
```

При `confidence < 0.85` — `clarify_question` заполнено строкой.

## Особые команды (без классификации)

| Команда | Действие |
|---------|----------|
| `/start` | Приветствие + справка |
| `/help` | Справка |
| `/график` | Ссылка на web-график |
| `/статус` | Короткий отчёт (= query_status) |
| `/отмена` | Отменить незавершённый диалог |
